# Housing Rental Service

Full-stack приложение для аренды жилья с REST API, авторизацией и пользовательским контентом

## Технологический стек

Frontend
- React 18
- TypeScript
- Redux Toolkit
- React Router
- Axios
- Vitest
- Testing Library

Backend
- Node.js
- Express
- Sequelize (ORM)
- PostgreSQL
- JWT (авторизация)
- Multer (загрузка файлов)
- Bcrypt (хеширование паролей)

Документация
- Swagger (OpenAPI 3.0)

## Требования для инструментов

- Node.js версии 18 или выше
- PostgreSQL версии 14 или выше
- npm версии 9 или выше

## Установка и запуск

1. Клонирование репозитория

Склонируйте проект и перейдите в корневую директорию.
```bash
    git clone https://github.com/StepanMuzafarov/react.git
    cd react
```
2. Настройка сервера

Перейдите в папку server и установите зависимости.
```bash
    cd server
    npm install
```
Создайте файл .env в папке server или измените его, добавьте следующие переменные окружения.
```env
    HOST=http://localhost
    PORT=5000
    DB_HOST=localhost
    DB_PORT=5432
    DB_NAME=rental_service
    DB_USER=postgres
    DB_PASSWORD=ваш_пароль_от_бд
    JWT_SECRET=ваш_секретный_ключ
```
Запустите сервер в режиме разработки.
```bash
    npm run dev
```
Сервер будет доступен по адресу http://localhost:5000

Документация Swagger доступна по адресу http://localhost:5000/api-docs

3. Настройка клиента

Перейдите в папку client и установите зависимости.
```bash
    cd ../client
    npm install
```
Запустите клиентское приложение.
```bash
    npm run dev
```
Приложение будет доступно по адресу http://localhost:5173

## Настройка базы данных

1. Создайте базу данных в PostgreSQL с именем rental_service.
```sql
    CREATE DATABASE rental_service;
```
2. При первом запуске сервера таблицы создадутся автоматически благодаря синхронизации Sequelize.

## Основные функции

Пользователи
- Регистрация с загрузкой аватара
- Авторизация через JWT-токен
- Проверка статуса авторизации
- Выход из системы

Предложения по аренде
- Просмотр списка предложений с фильтрацией по городу
- Детальная информация о предложении
- Загрузка 6 фотографий плюс превью
- Добавление и удаление из избранного
- Сортировка по популярности, цене и рейтингу

Отзывы
- Просмотр отзывов к предложению
- Добавление отзыва (только для авторизованных пользователей)
- Отображение данных автора (имя, аватар)

Работа с файлами
- Загрузка изображений через multipart/form-data
- Автоматическое сохранение в папку server/static
- Отдача статических файлов через Express

## Основные API эндпоинты
```
POST /register              - Регистрация пользователя
POST /login                 - Авторизация
GET  /login                 - Проверка токена
DELETE /logout              - Выход
GET  /offers                - Список предложений
GET  /offers/:id            - Детали предложения
POST /offers                - Создание предложения
GET  /favorite              - Избранные предложения
POST /favorite/:offerId/:status - Управление избранным
GET  /comments/:offerId     - Отзывы к предложению
POST /comments/:offerId     - Добавить отзыв
```

## Потенциальные ошибки и как их устранить

- Ошибка подключения к базе данных

Проверьте файл .env и убедитесь, что PostgreSQL запущен.

- Ошибка 401 при запросах

Убедитесь, что токен передан в заголовке Authorization: Bearer <token>.

- Не загружаются изображения

Проверьте права на запись в папку server/static.

- CORS ошибки

Убедитесь, что middleware cors подключен в index.js на сервере.

- Ошибки валидации

Проверьте, что данные соответствуют требованиям:
- Пароль: 6-12 символов
- Имя пользователя: 1-15 символов
- Фотографии оффера: ровно 6 файлов
